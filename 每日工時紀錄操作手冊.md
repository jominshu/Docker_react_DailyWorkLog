# 每日工時紀錄操作手冊

## 1. 系統概述
本系統用於記錄與統計每日支援工時，前端為 React，後端為 Go (Gin)，並透過 Oracle/ PostgreSQL 取得與保存資料。

## 2. 專案目錄重點
- `daily-work-log/`：前端原始碼
- `daily-work-log-go-backend/`：後端原始碼
- `daily-work-log-go-backend/web/build/`：前端靜態檔（後端實際提供的檔案）
- `docker-compose.yml`：容器啟動設定

## 3. 環境需求
- Docker / Docker Compose（正式部署建議）
- Node.js + npm（重新建置前端時需要）
- Go（本機開發後端時需要）

## 4. 環境變數設定（後端 `.env`）
位置：`daily-work-log-go-backend/.env`

- `POSTGRES_HOST`：PostgreSQL 主機
- `POSTGRES_PORT`：PostgreSQL 連線埠
- `POSTGRES_USER`：PostgreSQL 帳號
- `POSTGRES_PASSWORD`：PostgreSQL 密碼
- `POSTGRES_DB`：PostgreSQL 資料庫名稱
- `ORACLE_HOST`：Oracle 主機
- `ORACLE_PORT`：Oracle 連線埠
- `ORACLE_SERVICE`：Oracle 服務名稱
- `ORACLE_USER`：Oracle 帳號
- `ORACLE_PASSWORD`：Oracle 密碼
- `JWT_SECRET`：JWT 簽章密鑰（必填）
- `AUTH_COMPID`：Oracle 查詢用公司代碼
- `PORT`：後端服務啟動埠（預設 5000）
- `DEFAULT_ADMIN_EMPNO`：預設管理員工號（若不需要請留空）

## 5. 啟動方式

### 5.1 以 Docker 啟動（建議）
在專案根目錄執行：

```
docker compose up --build -d
```

查看容器狀態：

```
docker compose ps
```

查看後端日誌：

```
docker logs -f daily_work_log-backend-1
```

關閉服務：

```
docker compose down
```

瀏覽器開啟（依實際對外埠號）：
- `http://<主機IP>:19210`

### 5.2 本機開發啟動（前後端同時）
使用專案根目錄的腳本：

```
./dev-start.sh
```

預設前端：`http://localhost:3000`  
預設後端：`http://localhost:5000`

## 6. 使用操作流程

### 6.1 登入
使用 Oracle 工號與密碼登入，登入後系統會發放 JWT。

### 6.2 新增紀錄
1. 選擇支援公司（可多選）
2. 選擇日期
3. 填寫支援時數
4. 填寫支援內容
5. 點「新增」

### 6.3 歷史紀錄
- 可依日期區間篩選
- 可編輯或刪除自己的紀錄
- 可匯出 CSV

### 6.4 月報
- 選擇年份與月份（可選整年）
- 查詢後可匯出 CSV

### 6.5 支援時數匯總
- 選擇年份與月份（可選整年）
- 依公司彙整時數
- 可匯出 CSV

### 6.6 管理員設定
管理員可新增/編輯/刪除管理員清單。

### 6.7 權限管理
管理員可設定各員工可使用的功能分頁（新增紀錄/歷史紀錄/月報/匯總/管理員/權限管理）。

## 7. 前端更新與部署
前端更新後需重新 build 並同步到後端靜態目錄：

```
cd daily-work-log
npm run build
cp -r build/* ../daily-work-log-go-backend/web/build/
```

若使用 Docker，需重新建置映像：

```
docker compose up --build -d
```

## 8. 常見問題

### 8.1 無法連線資料庫
請確認 `.env` 的連線參數是否正確，並確保主機可連到公司內部 PostgreSQL/Oracle。

### 8.2 進入頁面顯示空白或舊版
請確認 `web/build` 是否為最新版本，並重新建置後端映像。

### 8.3 登入失敗
請確認 Oracle 帳密正確，並檢查後端是否連上 Oracle。

## 9. 備註
`.env` 內含敏感資訊，請勿提交至版本控制。
